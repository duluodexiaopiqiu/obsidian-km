## 一句话理解备份

**备份就是给你的知识库买保险** —— 你花了几百个小时写的笔记，一旦电脑坏了、文件误删、同步出错，没有备份就是灭顶之灾。备份做好了，哪怕电脑明天爆炸，你的知识库也能在另一台电脑上完整复活。

本文教你：**从零开始，找到适合自己的 Obsidian 备份方案，做到"丢不了、找得回、不折腾"。**

## 为什么必须备份

### 数据丢失的常见原因

<sheet sheet-id="7bwi2o" token="Thf6sDS53hlFtztTf7YcV04wn8O"></sheet>

> ⚠️ **一个残酷的事实**：没有备份的人，只有在数据丢失的那一刻，才会意识到备份的重要性。别等那一天。

### 备份的三个层次

一个好的备份方案，应该做到这三点：

1. **丢不了** — 本地硬盘坏了，数据还在别处（异地备份）
2. **找得回** — 文件被误改或误删，能恢复到之前的版本（版本历史）
3. **不折腾** — 全自动运行，不需要你每天手动操作（自动化）

## 备份前要知道的事

### 需要备份什么

Obsidian 的 Vault 就是一个普通文件夹，**整个文件夹都需要备份**，包括：

<sheet sheet-id="Jy6dhE" token="Thf6sDS53hlFtztTf7YcV04wn8O"></sheet>

> 💡 **小知识**：`.obsidian/` 文件夹是隐藏文件夹，里面存了你的所有 Obsidian 配置。备份它，换电脑时设置就不会丢。

### 备份的黄金法则

**3-2-1 原则**（备份领域的经典法则）：

- **3** 份数据副本（1 份原件 + 2 份备份）
- **2** 种不同存储介质（如：本地硬盘 + 云端）
- **1** 份异地备份（如：云盘、远程服务器）

> 新手不用一步到位，先做到"本地 + 云端"双备份，就已经超过 90% 的人了。

## 方案一：手动复制（最简单，但不推荐长期使用）

### 操作方法

1. 关闭 Obsidian
2. 找到你的 Vault 文件夹
3. 右键 → 复制
4. 粘贴到另一个位置（U 盘、移动硬盘、另一个文件夹）

### 优缺点

<sheet sheet-id="T6InHU" token="Thf6sDS53hlFtztTf7YcV04wn8O"></sheet>

> ⚠️ 手动复制只适合临时过渡，**千万不要作为唯一的备份方案**。

### 如果非要用，至少做到

1. 每周固定一天（比如周日晚上）执行备份
2. 备份到不同的物理设备（移动硬盘 / U 盘），不要备份到同一块硬盘的另一个分区
3. 保留最近 2-3 个版本的备份，用日期命名：`知识库备份_2026-06-25`

## 方案二：云盘同步（推荐新手首选）

### 原理

把 Vault 文件夹放在云盘的同步目录里，云盘客户端会自动把文件同步到云端。你修改笔记，云盘自动备份，全程无感。

### 常见云盘对比

<sheet sheet-id="OyLKP9" token="Thf6sDS53hlFtztTf7YcV04wn8O"></sheet>

> **国内用户首选坚果云**：同步稳定、支持 WebDAV、增量同步（只传修改的部分），免费版每月 1GB 上传流量对纯文本笔记绰绰有余。

### 实战：用坚果云同步 Obsidian

#### 第一步：注册并安装坚果云

1. 访问 [jianguoyun.com](https://www.jianguoyun.com/) 注册账号
2. 下载并安装坚果云客户端
3. 登录后，坚果云会创建一个同步文件夹（通常是 `~/Nutstore Files/`）

#### 第二步：把 Vault 移到同步文件夹

1. 关闭 Obsidian
2. 把你的 Vault 文件夹移动到坚果云同步目录下
3. 重新打开 Obsidian → 打开文件夹 → 选择新路径下的 Vault

> ⚠️ **注意**：是"移动"Vault 到同步文件夹，不是"复制"。否则你改的还是原件，同步的是副本，两边不一致。

#### 第三步：验证同步

1. 在 Obsidian 里新建一篇笔记，随便写点什么
2. 等待 10-30 秒，查看坚果云客户端是否有同步记录
3. 登录坚果云网页版，确认笔记已经出现在云端

### 云盘同步的注意事项

**问题 1：同步冲突**

多设备同时编辑同一篇笔记时，云盘会产生冲突文件（文件名带"冲突"字样）。

> **解决**：避免同时在两台设备打开同一篇笔记。如果不小心产生了冲突文件，对比两个版本，保留正确的，删掉另一个。

**问题 2：Obsidian 打开慢**

如果笔记很多（1000+ 篇），云盘实时同步会拖慢 Obsidian。

> **解决**：在坚果云设置中，把同步模式改为"按需同步"或"定时同步"，不要实时同步。

**问题 3：`.obsidian` 配置冲突**

不同设备的插件路径、主题可能不同，同步 `.obsidian` 文件夹可能导致配置错乱。

> **解决**：如果你在 Windows 和 Mac 之间同步，建议在坚果云中排除 `.obsidian/plugins/` 目录，每台设备单独装插件。如果只用同一系统（都是 Windows），则无需排除。

## 方案三：Git 版本控制（技术用户推荐）

### 原理

Git 是程序员用的版本控制工具。每次提交（commit）就像给笔记拍一张"快照"，你可以随时回到任何一个历史版本。配合 GitHub/Gitee 远程仓库，还能实现云端备份。

### 优缺点

<sheet sheet-id="ESxgx4" token="Thf6sDS53hlFtztTf7YcV04wn8O"></sheet>

### 实战：用 Git + GitHub 备份 Obsidian

#### 第一步：安装 Git

1. 访问 [git-scm.com](https://git-scm.com/) 下载安装
2. 安装时一路默认即可
3. 打开终端（Windows 用 Git Bash），验证安装：

```Bash
git --version
```

#### 第二步：创建 GitHub 仓库

1. 注册 [GitHub](https://github.com) 账号（国内可用 [Gitee](https://gitee.com) 替代）
2. 点击 `New repository`，创建一个新仓库
3. 仓库名随意（如 `my-knowledge-base`）
4. **不要勾选** "Add a README file"（保持仓库为空）
5. 选择 **Private**（私有仓库），保护你的笔记隐私
6. 点击 `Create repository`

#### 第三步：初始化 Vault 的 Git 仓库

打开终端，进入你的 Vault 文件夹：

```Bash
cd "E:/你的Vault路径"
```

初始化 Git：

```Bash
git init
git remote add origin https://github.com/你的用户名/你的仓库名.git
```

#### 第四步：创建 `.gitignore` 文件

在 Vault 根目录创建 `.gitignore` 文件，排除不需要版本控制的文件：

```Plain Text
# Obsidian
.obsidian/workspace
.obsidian/workspace.json
.obsidian/graph.json
.obsidian/cache
.trash/

# 系统
.DS_Store
Thumbs.db
desktop.ini

# 如果不想同步插件配置，取消下面注释
# .obsidian/plugins/
```

> 💡 排除 `workspace.json` 是因为它是 Obsidian 的工作区状态（打开了哪些标签页等），每次关闭都会变，频繁 commit 很烦。

#### 第五步：首次提交并推送

```Bash
git add .
git commit -m "初始化知识库"
git branch -M main
git push -u origin main
```

#### 第六步：日常使用

每次写完笔记，手动提交：

```Bash
git add .
git commit -m "更新笔记"
git push
```

> 觉得手动太麻烦？看下面的「Obsidian Git 插件」自动化方案。

#### 安装插件

1. Obsidian → 设置 → 社区插件 → 浏览
2. 搜索 `Obsidian Git`
3. 安装并启用

#### 配置自动备份

打开插件设置，按以下推荐配置：

<sheet sheet-id="mZ0nyE" token="Thf6sDS53hlFtztTf7YcV04wn8O"></sheet>

配置完成后，插件会在后台自动帮你 commit + push，**完全无感**。

> ⚠️ 首次使用前，需要先在插件设置中配置好 Git 用户名和邮箱，或者在终端执行：
> 
> ```Bash
> git config --global user.name "你的名字"

### Git 方案的版本回退

这是 Git 最强大的功能——回到任何一个历史版本：

```Bash
# 查看所有提交记录
git log --oneline

# 回到某个版本（不丢失后续记录）
git checkout <commit-hash>

# 回到最新版本
git checkout main
```

> 在 Obsidian Git 插件中，也可以通过命令面板（Ctrl/Cmd + P）搜索 "Obsidian Git: Open source control view" 来可视化查看历史。

### 原理

Obsidian 官方提供的同步服务，端到端加密，服务器看不到你的内容。和 Obsidian 深度集成，体验最好。

### 优缺点

- 对隐私要求高的人
- 不想折腾技术方案的人

### 如何开通

1. Obsidian → 设置 → 社区插件 → 浏览

## 如何恢复数据

备份的最终目的是"能恢复"。下面是各种方案的恢复方法：

### 从云盘恢复

1. 云盘网页版 / 客户端 → 找到误删的文件
2. 大多数云盘有「回收站」或「历史版本」功能
3. 找到需要的版本 → 下载 / 恢复

### 从 Git 恢复

```Bash
# 查看历史版本
git log --oneline

# 恢复某个文件到指定版本
git checkout <commit-hash> -- "文件路径"

# 整个 Vault 回到指定版本
git checkout <commit-hash>
```

### 从 Obsidian Sync 恢复

1. 右键笔记 → 「恢复到之前的版本」
2. 选择要恢复的时间点
3. 确认恢复

### 从手动备份恢复

1. 找到备份文件夹
2. 复制所有文件
3. 粘贴到 Vault 目录（覆盖现有文件）

> ⚠️ 恢复前先备份当前状态！万一恢复出错，还能回到恢复前。

## 常见问题

### 问题 1：Git 提交时报错 "Author identity unknown"

**原因**：没有配置 Git 用户信息。

**解决**：在终端执行：

```Bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱"
```

### 问题 2：坚果云同步时 Obsidian 卡顿

**原因**：坚果云实时监控文件变化，笔记多时频繁触发同步。

**解决**：

- 坚果云设置 → 把同步模式改为「定时同步」（如每 30 分钟）
- 或在坚果云中把 `.obsidian/` 目录设为忽略

### 问题 3：Git push 报错 "src refspec main does not match"

**原因**：默认分支名不是 `main`，可能是 `master`。

**解决**：

```Bash
# 查看当前分支名
git branch

# 如果是 master，重命名为 main
git branch -M main

# 再次推送
git push -u origin main
```

### 问题 4：云盘同步后插件配置丢失

**原因**：不同设备的路径不同，`.obsidian/plugins/` 里的配置不通用。

**解决**：

- 在云盘设置中排除 `.obsidian/plugins/` 目录
- 每台设备单独安装插件，手动配置
- 或者：接受配置可能不一致，以一台设备为主

### 问题 5：GitHub 推送速度很慢 / 超时

**原因**：国内访问 GitHub 网络不稳定。

**解决**：

- 使用 [Gitee](https://gitee.com) 替代 GitHub（国内速度快）
- 或配置 Git 代理（如果你有代理工具）：

```Bash
git config --global http.proxy http://127.0.0.1:端口号
git config --global https.proxy http://127.0.0.1:端口号
```

## 给新手的建议

1. **今天就做备份，不要等"明天"** —— 数据丢失不会提前通知你。
2. **先有备份，再优化备份** —— 哪怕只是手动复制到 U 盘，也比没有强。先跑起来，再慢慢优化。
3. **定期验证备份是否有效** —— 每隔几个月，尝试从备份恢复一个文件，确保备份没有损坏。一个恢复不了的备份等于没有备份。
4. **不要只依赖一种方案** —— 云盘 + Git、云盘 + 移动硬盘，多一层保险多一份安心。
5. **图片和附件也要备份** —— 很多人只备份了 `.md` 文件，忘了 `6.assets/` 里的图片，结果笔记里的图片全裂了。

## 小练习

1. 检查你当前的 Vault 是否有任何形式的备份，如果没有，今天就选一个方案开始。
2. 如果选择云盘方案：注册坚果云，把 Vault 移到同步文件夹，验证同步是否生效。
3. 如果选择 Git 方案：安装 Git，创建 GitHub/Gitee 仓库，完成首次提交并推送。
4. 故意删掉一篇不重要的笔记，然后从备份中恢复它，验证你的备份方案是否真的有效。
5. 在日历上设一个每月的"备份检查日"提醒，定期验证备份的完整性。
