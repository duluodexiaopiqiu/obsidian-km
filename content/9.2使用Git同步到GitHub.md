# 9.2 使用 Git 同步到 GitHub（数据备份与多端同步）

## 为什么要用 Git 同步？

Obsidian 的笔记是本地文件，这意味着：
- 你电脑坏了 → 笔记全丢
- 你换了电脑 → 笔记过不来
- 想在公司接着写 → 不行

**Git + GitHub** 就是用来解决这些问题的。Git 是版本管理工具，GitHub 是云端托管平台。搭配使用，你的笔记就能：

- **自动备份**到云端
- **多设备同步**（家里、公司、笔记本）
- **版本回溯**（误删了还能找回）

## 前置准备

### 需要安装的软件
1. **Git**：https://git-scm.com/downloads
2. **GitHub 账号**：https://github.com 免费注册
3. **Obsidian Git 插件**（可选，用于在 Obsidian 中直接操作）

### 检查 Git 是否安装
打开终端（CMD / PowerShell / 终端），输入：
```bash
git --version
```
如果显示版本号就说明已安装。

## 操作步骤

### 第一步：在 GitHub 上创建仓库
1. 登录 GitHub，点击右上角"+" → "New repository"
2. 仓库名称：`obsidian-vault`（或你喜欢的名字）
3. 选择 **Private**（私人仓库，别人看不到）
4. 不要勾选"Initialize this repository with a README"
5. 点击"Create repository"

### 第二步：初始化本地仓库
打开终端，进入你的 Obsidian Vault 目录：

```bash
cd 你的知识库文件夹路径
git init
```

### 第三步：创建 .gitignore 文件
在 Vault 根目录创建 `.gitignore` 文件，写入：

```
# Obsidian 配置（不同步）
.obsidian/workspace
.obsidian/cache
.obsidian/plugins/*/data.json

# 系统文件
.DS_Store
Thumbs.db
```

> **注意**：不把 `.obsidian` 完全忽略，因为插件配置还是需要同步的，只忽略工作区状态和缓存。

### 第四步：提交并推送到 GitHub

```bash
git add .
git commit -m "初始化 Obsidian 知识库"

# 关联远程仓库（用你自己的仓库地址）
git remote add origin https://github.com/你的用户名/obsidian-vault.git

# 推送到 GitHub
git push -u origin main
```

### 第五步：在其他设备上同步

在其他电脑上：

```bash
git clone https://github.com/你的用户名/obsidian-vault.git
```

然后用 Obsidian 打开这个文件夹即可。

## 日常使用流程

### 方式一：手动操作（推荐新手）
每次写完笔记后：

```bash
git add .
git commit -m "更新笔记"
git push
```

到另一台设备上：

```bash
git pull
```

### 方式二：Obsidian Git 插件（一键同步）
1. 安装社区插件 **Obsidian Git**
2. 设置自动提交间隔（如每30分钟）
3. 通过快捷键或命令面板一键"提交并推送"

## 常见问题

### Q：同步冲突了怎么办？
如果两台设备都修改了同一篇笔记，Git 可能会报冲突。解决方式：
1. 用 VS Code 或 GitHub Desktop 打开冲突文件
2. 选择保留哪个版本
3. 提交解决后的版本

### Q：笔记包含敏感信息怎么办？
- **私人仓库** —— GitHub 的私人仓库是加密的
- **自托管** —— 也可以用 GitLab 或自己搭建 Gitea 服务
- **加密** —— 某些敏感笔记可以用 Obsidian 加密插件

### Q：同步不成功？
常见原因：
- 忘记 `push` —— 只 commit 了但没有推送到 GitHub
- 忘记 `pull` —— 另一台设备有更新但没有拉取
- 网络问题 —— GitHub 在国内可能访问不稳定，可以配置代理或使用 Gitee

## 进阶：自动同步流

```
写笔记 → Obsidian Git 自动 commit + push → GitHub（云端备份）
                                         → 另一台设备自动 pull（保持最新）
```

配合 **Obsidian Git 插件** 的自动备份功能，几乎可以做到无感同步。

## 小练习

1. 注册 GitHub 账号（如果还没有）
2. 创建一个私人仓库
3. 把你的 Vault 推送到 GitHub
4. 在 Obsidian 中安装并配置 Obsidian Git 插件
5. 确保自动提交正常运行
