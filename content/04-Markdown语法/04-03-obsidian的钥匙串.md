# Obsidian「钥匙串（密钥）」完整讲解

## 一、这个功能是干嘛的？

图里左侧菜单**钥匙串**，就是 Obsidian 内置的加密密钥管理器，官方用来安全存放各类插件的敏感凭证：

- 各类 AI 大模型 API Key（OpenAI、DeepSeek、通义千问、Claude 等）
- 云服务账号密码、同步工具 Token、图床密钥、网盘授权码
- 第三方服务登录凭证、数据库密钥等隐私信息

### 核心优势：安全隔离

1. **不明文存在笔记库文件里**

如果你直接把 API Key 写在 md 笔记里，同步网盘、分享库时很容易泄露；钥匙串是独立加密存储，不会暴露在库内文本中。

1. **系统级加密保护**

Windows/macOS/Linux/移动端都会调用系统自带密钥加密服务（Windows凭据管理器、mac钥匙串访问、手机安全加密区），凭证是加密存放，不会裸漏。

1. **仅插件有权限读取**

插件只能读取你授权给它的密钥，其他插件看不到这条密钥；普通笔记、搜索完全读不到密钥内容。

## 二、页面提示「尚未添加密钥」是什么意思？

你现在库里面**没有任何插件存储过密钥**，所以页面空白。

只有当你安装了需要 API/密码 的插件，在插件设置里填入密钥时，这条密钥才会自动出现在这个「钥匙串」面板。



举个常见场景：

装了 AI 写作插件（比如 Text Generator、Copilot），填入 DeepSeek API Key，保存后，这个 Key 就会出现在钥匙串列表里。



## 三、钥匙串面板能做什么操作

当有密钥条目后，每条会显示：服务名称、绑定的插件、创建时间，支持操作：

1. **查看**：临时展示完整密钥（仅当前窗口可见，复制后记得清空剪贴板）
2. **复制**：一键复制密钥
3. **删除**：移除这条密钥，对应插件会立刻失去该凭证，需要重新填写
4. **重命名**：给密钥备注区分，比如「DeepSeek-个人专用」「阿里云图床」

## 四、和直接写在笔记里的区别（重点）

| 方式 | 安全性 | 同步风险 | 泄露概率 |
|-|-|-|-|
| 钥匙串存储 | 高，系统加密，不进库文件 | 同步库完全不带密钥 | 极低 |
| 写在MD笔记 | 明文文本，存在库文件夹 | 网盘/多端同步会同步密钥 | 极高，误分享、仓库泄露直接丢Key |



## 五、常见使用场景

1. AI 生成类插件：各类大模型 API Key
2. 图床插件：PicGo、Imgur、阿里云OSS AccessKey
3. 同步/备份插件：WebDAV、OneDrive、Google Drive 账号密码
4. 翻译插件：百度翻译、DeepL 开发者密钥
5. 数据库/API 联动插件：第三方服务 Token

## 六、安全小提示

1. 不要把密钥截图、复制到普通笔记；
2. 换电脑使用库时，钥匙串不会同步（加密绑定本机系统），新设备需要重新填写插件密钥；
3. 卸载插件不会自动删除钥匙串条目，不用的密钥记得手动删除；
4. 如果你共用电脑，离开前关闭 Obsidian，避免他人查看钥匙串里的密钥。

## 七、补充：为什么你现在是空的？

你的库目前没有任何需要填写 API/密码 的第三方插件写入凭证。

只要你打开任意需要密钥的插件设置，填入密钥并保存，这里就会出现对应的密钥条目。
