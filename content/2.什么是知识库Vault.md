# 2. 什么是知识库（Vault）—— Obsidian 的核心概念

## 一句话理解 Vault

**Vault（库）就是一个文件夹**，里面放着你的所有笔记文件（`.md` 格式）。Obsidian 打开这个文件夹，读取里面的所有文件，就构成了你的"知识库"。

## Vault 的三大特点

### 1. 本地存储
你的笔记就是普通的 Markdown 文件，存在你自己指定的文件夹里。不需要联网，不需要云端同步。

```
我的知识库/
├── 笔记1.md
├── 笔记2.md
├── 项目A/
│   ├── 需求分析.md
│   └── 技术方案.md
└── 日记/
    ├── 2026-06-01.md
    └── 2026-06-02.md
```

### 2. 纯文本格式
所有笔记都是 `.md` 文件（Markdown 格式），这是最通用的文本格式。即使有一天你不想用 Obsidian 了，你的笔记仍然可以用任何文本编辑器打开。

### 3. 一个 Vault = 一个知识体系
你可以：
- 把所有笔记放在**一个 Vault** 里 —— 知识互联互通
- 为不同用途创建**多个 Vault** —— 工作和生活分开

## 第一次创建 Vault

打开 Obsidian 后，你会看到两个选项：

### 选项 A：创建新库
1. 点击"创建新库"
2. 输入库名称（比如"我的知识库"）
3. 选择存放位置（建议放在容易备份的地方）
4. 点击"创建"

### 选项 B：打开已有文件夹
如果你已经有一些 Markdown 文件，直接选择"打开本地文件夹"，选中那个文件夹即可。

## Vault 里还能放什么？

除了笔记文件，Vault 里还可以放：

- **附件**：图片、PDF、音频文件
- **模板文件**：笔记模板（后面会讲）
- **插件配置**：社区插件的设置数据
- `.obsidian` 文件夹：Obsidian 自动生成的配置目录（**不要手动删除**）

```
我的知识库/
├── .obsidian/          ← Obsidian 配置（自动生成）
├── 笔记/
│   ├── 学习笔记.md
│   └── 工作笔记.md
├── 附件/
│   ├── 图片1.png
│   └── 文档.pdf
├── 模板/
│   └── 日记模板.md
└── 日记/
    └── 2026-06-17.md
```

## 给新手的建议

1. **先从一个 Vault 开始**，把所有笔记放进去，让知识自然连接
2. **不要把 Vault 放在云同步文件夹（如 iCloud、OneDrive）的根目录**，可能会引起同步冲突
3. **定期备份** —— 可以用 Git 或手动拷贝，数据无价
4. **给 Vault 起个好名字**，你会用很久

## 小练习

1. 在桌面新建一个文件夹，命名为"我的第一个知识库"
2. 用 Obsidian 打开它
3. 在 Obsidian 里新建一个文件，写入"Hello Obsidian"，保存
4. 去文件夹里看看，是不是多了一个 `.md` 文件？
