## 一句话理解 Web Clipper

**Obsidian Web Clipper 是官方浏览器插件**，一键把网页内容剪藏到你的 Vault 中。就像给浏览器装了一个"复制按钮"，看到好文章直接"丢"进 Obsidian，不用手动复制粘贴。

结合 AI，它还能**自动提取重点、分类打标、生成摘要**，让收录从"搬运工"变成"智能管家"。

## 安装 Obsidian Web Clipper

### 第一步：安装浏览器插件

1. 打开 Chrome/Edge/Firefox 应用商店
2. 搜索 **"Obsidian Web Clipper"**
3. 点击安装（认准官方出品，开发者是 Obsidian Team）

> 国内用户如果访问不了 Chrome 商店，可以：
> 
> - 用 Edge 浏览器，在 Edge 插件商店搜索安装
> - 或者去 GitHub 下载 `.zip` 包手动安装

### 第二步：连接你的 Vault

1. 安装完成后，点击浏览器右上角的 Web Clipper 图标
2. 选择"连接 Obsidian"
3. 插件会自动检测你本地的 Vault 列表，选择你要保存的 Vault
4. 完成授权

> 如果你的 Vault 没有出现在列表里，点击"手动输入路径"，粘贴 Vault 的完整文件夹路径。

## 基础用法：快速剪藏网页

### 模式一：全文剪藏（默认）

看到一篇好文章，点击 Web Clipper 图标 → 选择"全文" → 点击保存。

- 自动提取网页正文
- 去掉广告、导航栏、推荐内容
- 保留图片和链接

### 模式二：选中剪藏

只想保存网页的一部分？

1. 鼠标选中你要保存的文字
2. 点击 Web Clipper 图标 → 选择"选中文本"
3. 只保存你选中的内容

### 模式三：书签模式

只想保存链接和标题？

- 点击 Web Clipper → 选择"书签"
- 只保存标题 + URL + 一小段摘要
- 适合"先标记，以后再看"的场景

## 进阶用法：结合 AI 进行智能收录

### 什么是"智能收录"

不是简单把网页内容复制进来，而是让 AI 帮你：

1. **提取核心观点** — 去掉冗余，保留干货
2. **自动分类打标** — 根据内容自动打上标签
3. **生成个人摘要** — 用你自己的话重写一遍（费曼学习法）
4. **关联已有笔记** — 找出和现有笔记的关联，建立双链

### 工作流：一键智能收录

#### 步骤 1：设置 Web Clipper 模板

在 Web Clipper 设置中，创建一个自定义模板：

```Markdown
---
type: clipper
source: {{url}}
date: {{date}}
tags: 待整理
---

# {{title}}

> 原文链接：[{{title}}]({{url}})
> 收录时间：{{date}}

## 原文摘要

{{content}}

## AI 提取的核心观点

（粘贴 AI 分析结果）

## 我的思考

-

## 行动清单

- [ ] 

## 关联笔记

- [[]]
```

#### 步骤 2：剪藏后调用 AI 处理

保存到 Obsidian 后，选中笔记内容，调用 AI（如 Kimi、ChatGPT、DeepSeek）：

> "请帮我提取这篇文章的 3-5 个核心观点，用简洁的中文总结，并建议 2-3 个分类标签。"
> 
> "这篇文章和我现有的笔记有什么关联？请帮我找出可以建立双链的地方。"
> 
> "请用我自己的话重写这篇文章的核心内容，让它更容易理解，适合作为我的知识库笔记。"

#### 步骤 3：回填到笔记

把 AI 的分析结果粘贴回笔记的对应区域，完成"智能收录"。

## 实战场景：三种典型的收录工作流

### 场景 1：每天浏览文章，做知识积累

**适用人群**：自媒体创作者、知识工作者、终身学习者

**工作流**：

1. 早上刷公众号/知乎/少数派，看到好文章
2. 点击 Web Clipper → 全文保存到 `4.收件箱/`
3. 晚上统一处理：打开收件箱，每篇文章调用 AI 提取核心观点
4. 把处理好的文章移到 `3.学习笔记/` 或 `9.读书/`，打好标签
5. 周末回顾，建立双链，形成知识网络

> **关键**：不要看一篇处理一篇，先集中"丢"进收件箱，再统一处理，效率更高。

### 场景 2：做竞品调研/行业分析

**适用人群**：产品经理、运营、市场人员

**工作流**：

1. 搜索竞品文章、行业报告、新闻资讯
2. Web Clipper 保存到 `4.收件箱/竞品调研/`
3. 用 AI 提取：竞品做了什么功能、用了什么策略、数据表现如何
4. 在 Obsidian 中建立对比表格，形成自己的分析框架
5. 输出调研报告

> **模板提示**：在 Web Clipper 中设置一个"竞品调研"专用模板，自动包含"竞品名称、功能分析、优缺点、可借鉴点"等字段。

### 场景 3：收集写作素材

**适用人群**：内容创作者、博主、写作者

**工作流**：

1. 浏览时看到金句、案例、数据，随时用 Web Clipper 保存
2. 保存时选择"选中文本"模式，只保存关键段落
3. 用 AI 标注：这条素材适合什么主题？可以用在什么场景？
4. 在 Obsidian 中建立素材库，按主题分类
5. 写作时直接搜索调用

> **技巧**：给素材打标签时，同时打上"#素材/金句"、"#素材/案例"、"#素材/数据"，方便分类检索。

## 常见问题与解决

### 问题 1：网页内容提取不完整

**症状**：保存后图片缺失、格式错乱

**解决**：

- 在 Web Clipper 设置中，开启"下载图片到本地"
- 如果还是不行，尝试用"选中文本"模式，手动选中正文区域
- 对于特殊网页（如微信公众号），可以先用"阅读模式"插件（如 Immersive Reader）净化页面，再剪藏

### 问题 2：AI 提取的内容质量不高

**症状**：AI 总结的摘要太笼统，没有抓到重点

**解决**：

- 给 AI 更具体的指令，比如： 

> - "请提取这篇文章的**可操作方法**，忽略背景介绍和理论阐述。"  
>  "请总结这篇文章的**反常识观点**，列出 3 个。"  
>  "请用 **bullet points** 列出文章的所有关键数据。"

- 如果文章太长，先让 AI 分段处理，再综合

### 问题 3：收录太多，整理不过来

**症状**：收件箱堆了 100 篇文章，越积越多，越来越焦虑

**解决**：

- 设置"每周清理日"，固定时间处理收件箱
- 使用"两分钟原则"：如果一篇文章 2 分钟内能处理完，当场处理；处理不完，打标签后移入待办
- 不要追求"每篇都精读"，有些文章提取 3 个要点就够了，存太多反而分散注意力

### 问题 4：Obsidian 里搜索不到网页内容

**症状**：明明保存了，但搜索关键词找不到

**解决**：

- 确保 Web Clipper 设置中开启了"提取全文"
- 如果保存的是图片或 PDF，Obsidian 默认不索引图片中的文字，需要配合 OCR 插件
- 给笔记打好标题和标签，方便检索

## 给新手的建议

1. **先养成"看到就收藏"的习惯** —— 不要因为"还没整理好体系"就不敢收藏，先存下来，再慢慢处理。
2. **收件箱是缓冲区，不是终点站** —— 定期清理收件箱，把内容移到对应的分类目录，否则会变成垃圾堆。
3. **AI 是助手，不是替代** —— AI 提取的内容一定要经过你的审核和修改，加入自己的思考，知识才能真正内化成你的。
4. **不要追求"完美收录"** —— 有些文章提取 3 个要点就够了，不要逼自己把每篇文章都写成完整笔记。
