# 6. 知识组织：文件夹、标签、链接，怎么选？

这是 Obsidian 新手问得最多的问题。笔记越写越多之后，如何组织才能不乱？

Obsidian 提供了三种组织方式：**文件夹**、**标签**、**链接**。它们各有优劣，最佳实践是**组合使用**。

## 方式一：文件夹（层级结构）

### 是什么
和电脑上的文件夹一样，把笔记按类别放在不同目录下。

```
知识库/
├── 工作/
│   ├── 项目A.md
│   └── 项目B.md
├── 学习/
│   ├── Python/
│   │   ├── 基础语法.md
│   │   └── 进阶.md
│   └── Obsidian/
│       └── 入门指南.md
└── 生活/
    ├── 旅行计划.md
    └── 读书笔记.md
```

### 优点
- **直观** —— 符合电脑使用习惯
- **结构清晰** —— 一目了然
- **适合归档** —— 回顾时方便浏览

### 缺点
- **死板** —— 一篇笔记只能放在一个文件夹
- **难以跨类别** —— 比如一篇笔记既涉及"Python"又涉及"工作"，只能二选一

### 适用场景
- 项目文档、课程笔记、日记等有明显归属的内容

## 方式二：标签（Tag）

### 是什么
给笔记打上关键词标签，类似微博/Twitter 的话题。

```markdown
# Python 基础语法

#编程/Python #教程 #新手

Python 是一种...
```

### 标签的层级
```
#编程            ← 宽泛标签
#编程/Python     ← 子标签（自动归入 #编程 下）
#编程/Python/基础 ← 更细分的子标签
```

### 如何添加标签
- 在笔记中直接写 `#标签名`
- 或在文件属性（Frontmatter）中定义：

```markdown
---
tags:
  - 编程
  - Python
  - 教程
---
```

### 标签面板
左侧栏的"标签面板"会列出你所有使用过的标签及使用次数，点击即可筛选出所有包含该标签的笔记。

### 优点
- **灵活** —— 一篇笔记可以有多个标签
- **跨越文件夹** —— 不同类别的笔记可以通过相同标签关联
- **筛选方便** —— 点一个标签就能找到所有相关笔记

### 缺点
- **容易失控** —— 标签越用越多，最后自己都忘了有哪些
- **标准难统一** —— 今天用"编程"，明天用"代码"，后天用"coding"

### 适用场景
- 跨类别的内容分类、状态标记（#待读 / #进行中 / #已完成）

## 方式三：链接（双向链接）

### 是什么
用 `[[笔记名]]` 把相关笔记连接起来。

### 优点
- **最灵活** —— 任意笔记可以链接任意笔记
- **自动发现** —— 反向链接让你发现没想到的关联
- **图谱驱动** —— 链接越多，知识网络越有价值

### 缺点
- **需要有内容才能链接** —— 新建的孤零零的笔记没有链接
- **浏览门槛高** —— 不像文件夹那样可以挨个浏览

### 适用场景
- 核心组织方式，所有笔记都应该被链接

## 三种方式如何组合？

### 推荐策略：文件夹（大分类）+ 标签（维度）+ 链接（血脉）

```
┌─────────────────────────────────────────┐
│                                         │
│  文件夹 → 大的分类（归档导向）             │
│  标签   → 维度标记（状态/类型/领域）        │
│  链接   → 知识关联（让笔记互联互通）         │
│                                         │
└─────────────────────────────────────────┘
```

### 我的建议

1. **文件夹不超过3层** —— 太深了反而不方便
2. **标签保持在20个以内** —— 太多就失去意义了
3. **每篇笔记至少链接1-2篇其他笔记** —— 这是让知识库"活"起来的关键

### 具体方案

```
知识库/
├── 0-收件箱/         ← 临时存放，还没分类的笔记
├── 1-项目/           ← 工作中的项目笔记
├── 2-领域/           ← 学习领域（编程、设计、写作等）
├── 3-资源/           ← 收藏的文章、工具、书籍
├── 4-日记/           ← 每日记录
├── 5-归档/           ← 已完成的项目
└── 6-模板/           ← 笔记模板
```

这个结构叫 **PARA 方法** 的变体（项目-领域-资源-归档），加上收件箱和日记，基本能满足大多数人的需求。

## 小练习

1. 创建上面推荐的文件夹结构
2. 把前面几节课写的笔记移动到对应的文件夹
3. 给每篇笔记添加 1-2 个标签
4. 检查笔记之间的链接，不足的补充上
