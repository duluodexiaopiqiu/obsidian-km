## 一、新增一篇笔记

### 步骤

1. 在 `content/` 目录下新建一个 `.md` 文件

```
content/
├── 11.实战项目一笔记方法.md    ← 新建的笔记
└── ...
```

2. 在文件顶部写标题，然后正常编写 Markdown 内容

```markdown
# 实战项目一：笔记方法

## 什么是卡片笔记法

卡片笔记法（Zettelkasten）是一种...

## 具体操作步骤

1. 第一步：...
2. 第二步：...
```

3. 如果想让它在**首页导航**中出现，编辑 `content/index.md`，加一行双向链接：

```markdown
11. [[11.实战项目一笔记方法]]
```

### 命名建议

| 推荐 ✅ | 避免 ❌ | 原因 |
|---------|---------|------|
| `11.标题.md` | `11 标题.md` | 用点号分隔编号，不要用空格 |
| `插件进阶用法.md` | `插件进阶用法 ` | 文件名末尾不要有多余空格 |
| `template-模板.md` | `my note.md` | 避免空格（URL 会变成 `%20`） |

支持**中文文件名**，完全没问题。

## 二、修改已有笔记

直接用编辑器打开 `content/` 下的对应 `.md` 文件，修改后保存即可。

```
content/4.掌握Markdown语法.md    ← 直接编辑这个文件
```

修改完成后，按 [九、发布到线上](#九发布到线上) 的步骤推送即可。

---

## 三、新增图片/附件

### 方法一：放到 content 同级或子目录（推荐）

```
content/
├── assets/
│   ├── obsidian-logo.png      ← 图片放这里
│   └── screenshot-demo.png
├── 1.Obsidian介绍.md
└── ...
```

在 Markdown 中引用（使用相对路径）：

```markdown
![Obsidian Logo](assets/obsidian-logo.png)
```

### 方法二：放到外部图床

```markdown
![截图](https://example.com/image.png)
```

### 注意事项

- **不要用** `![[image.png]]` 这种 Obsidian 专用语法，Quartz 不解析
- **要用标准 Markdown**：`![描述](路径)` 或 `![描述](URL)`
- 图片文件会被自动复制到构建产物中，无需手动处理

---

## 四、删除笔记

1. 直接删除 `content/` 下的 `.md` 文件
2. 如果首页有指向它的链接，记得从 `content/index.md` 中移除
3. 其他笔记中如果有 `[[该笔记]]` 的双向链接，也建议清理

---

## 五、组织目录结构

支持创建**子目录**来组织内容。例如按模块分组：

```
content/
├── index.md
├── 01-入门/
│   ├── Obsidian介绍.md
│   ├── 什么是知识库Vault.md
│   └── 安装下载与界面熟悉.md
├── 02-进阶/
│   ├── 掌握Markdown语法.md
│   ├── 创建第一篇笔记与双向链接.md
│   └── 知识组织文件夹标签链接.md
└── 03-高级/
    ├── 插件生态介绍与推荐安装.md
    └── 如何在Obsidian中使用AI.md
```

**改目录后记得**：
- 更新 `content/index.md` 中的链接路径（如果文件名变了）
- 更新其他笔记中的 `[[双向链接]]`（Quartz 会按文件名匹配，只要文件名不变，链接自动生效）

## 六、使用双向链接

Quartz 原生支持 Obsidian 的 `[[双向链接]]` 语法，这是本站点的核心特性。

### 链接到其他笔记

```markdown
想了解更多，请看 [[2.什么是知识库Vault]] 这篇笔记。

也可以 [[2.什么是知识库Vault|用自定义文字链接]]。
```

### 链接到笔记中的某个标题

```markdown
参考 [[4.掌握Markdown语法#列表语法]] 章节的内容。
```

### 链接到笔记中的某个段落（块引用）

在目标笔记中，给段落加上 `^标识符`：

```markdown
这是一段重要内容 ^important-note
```

在其他笔记中引用：

```markdown
详见 [[4.掌握Markdown语法#^important-note]]
```

### 反向链接（自动生成）

每篇笔记底部会**自动显示**所有引用了它的其他笔记，无需手动维护。

### 图谱视图

网站左上角有 **Graph（图谱）** 按钮，点击可看到所有笔记之间的链接关系网络图。

## 七、Front Matter 元数据（可选）

可以在 Markdown 文件顶部添加 YAML 格式的元数据，控制页面的显示行为：

```markdown
---
title: 自定义页面标题（不填则用第一个 # 标题）
description: 这篇笔记的描述（用于搜索结果和 SEO）
date: 2026-06-18
tags:
  - obsidian
  - 教程
draft: true
---

# 正文内容
```

### 常用字段

| 字段 | 作用 | 示例 |
|------|------|------|
| `title` | 页面标题（覆盖 `# 标题`） | `title: "我的笔记"` |
| `description` | 页面描述（搜索结果展示） | `description: "Obsidian 入门教程"` |
| `date` | 发布日期 | `date: 2026-06-18` |
| `tags` | 标签（自动生成标签页） | `tags: [obsidian, 教程]` |
| `draft` | 设为 `true` 则构建时自动跳过该文件 | `draft: true` |

## 八、本地预览

在推送之前，建议先在本地预览确认效果。

```bash
# 进入项目目录
cd E:\自媒体\知识管理\14.知识管理结合obsidian实战

# 安装依赖（只需首次执行）
npm install

# 启动本地预览
node quartz/bootstrap-cli.mjs build --serve
```

浏览器打开 **http://localhost:8080** 即可预览。

- 修改 `content/` 下的文件后，**刷新浏览器**即可看到更新
- `Ctrl + C` 可停止预览服务器

## 九、发布到线上

### 完整命令（三步走）

```bash
# 1. 添加所有变更到暂存区
git add -A

# 2. 提交（写清楚改了什么）
git commit -m "新增：第11章 实战项目"

# 3. 推送到 GitHub（自动触发部署）
git push
```

### 验证部署

1. 推送后打开 https://github.com/duluodexiaopiqiu/obsidian-km/actions
2. 最新的 "Deploy Quartz site to GitHub Pages" 应显示为绿色 ✅
3. 等 1-2 分钟后访问 https://duluodexiaopiqiu.github.io/obsidian-km/ 查看更新

### 快捷方式（一行搞定）

```bash
git add -A && git commit -m "更新笔记" && git push
```

## 十、常见问题

### Q：推送时提示 "Please tell me who you are"

说明 git 没有配置用户信息，执行：

```bash
git config user.name "你的名字"
git config user.email "你的邮箱"
```

### Q：本地预览报错 "Cannot find module"

重新安装依赖：

```bash
rm -rf node_modules package-lock.json
npm install
```

### Q：新增的笔记在网站上找不到

检查：
1. 文件是否放在 `content/` 目录下（不在其他位置）
2. 文件名是否以 `.md` 结尾
3. 文件是否设置了 `draft: true`（草稿状态不会发布）
4. 是否已经 `git push` 触发了部署

### Q：双向链接变成了红色/404

1. 确认目标文件确实存在于 `content/` 目录
2. 确认 `[[链接名]]` 和目标文件名**完全一致**（包括后缀省略）
3. 如果文件名含空格，链接里也要有空格：`[[my note]]` 对应 `my note.md`

### Q：想改网站标题/主题/颜色

编辑项目根目录下的 `quartz.config.ts`，修改 `configuration` 部分：

```typescript
configuration: {
  pageTitle: "你的网站标题",    // 网页标题
  locale: "zh-CN",             // 界面语言
  // theme.colors 里改配色...
}
```

修改后需要重新构建和推送。

### Q：想换域名（比如用自定义域名）

1. 购买域名并配置 DNS（CNAME 指向 `duluodexiaopiqiu.github.io`）
2. 在 `content/` 目录下创建 `CNAME` 文件，内容写你的域名
3. 在 GitHub 仓库 Settings → Pages → Custom domain 填入域名
4. 修改 `quartz.config.ts` 中的 `baseUrl` 为你的自定义域名
